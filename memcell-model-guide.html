How the Algorithm Works

MemCell processes data through a sequential pipeline. Each stage removes one class of redundancy that the others cannot. Bit-Pack and Byte Shuffle are ML-specific stages added in v0.2.0.

The Analyzer (Stage 1) automatically selects which downstream stages activate based on entropy and distinct byte count. High-entropy data skips expensive stages that cannot help. Every stage validates its own output — none can make data larger.

Stage-by-Stage Algorithm Reference

Stage
What it does
ML relevance
Skip condition
1 · Analyze
Reads 64 KB sample; computes Shannon entropy H = −Σ pᵢ log₂(pᵢ); builds byte histogram; classifies data type
Detects INT4 (16 distinct values → triggers BITPACK). Classifies float arrays as BINARY_STRUCTURED → enables SHUFFLE.
Never skipped
2 · Chunk
Buzhash rolling hash splits at natural boundaries. Target 4 KB, min 1 KB, max 16 KB.
Multi-layer models repeat tensors. Content-defined boundaries maximise deduplication hit rate across similar layers.
Always runs
3 · Dedup
FNV-1a 128-bit hash per chunk. Duplicates replaced by 4-byte reference ID instead of ~4 KB data.
Shared embedding layers, identical attention heads, checkpoints with unchanged layers — up to 50× for repeated structure.
Always runs
4 · Bit-Pack ML
Counts distinct values per chunk; packs to minimum bits. ≤2→1 bit, ≤4→2 bits, ≤16→4 bits, ≤64→6 bits.
INT4 weights: 16 values → 4 bits each → immediate 2× reduction before any other stage. INT8 with limited range benefits similarly.
Skipped if header overhead exceeds savings
5 · Byte Shuffle ML
Transposes byte planes: groups all byte-0s, byte-1s, etc. from typed arrays. [A0 A1 A2 A3][B0...] → [A0 B0 C0...][A1 B1 C1...]
Float32 byte 0 (sign+exponent MSB) is nearly constant across weights. Grouping creates long runs LZSS compresses 30–50% better.
Skipped if no element size reduces entropy by ≥0.5 bits/byte
6 · Pattern Genome
Builds 255-entry codebook of frequent 2–8 byte sequences; replaces them with escape+code pairs. O(1) hash lookup.
Common weight prefixes and tensor header patterns captured. Codebook serialized in .cell file for decoder.
Skipped if output ≥ input
7 · Delta
XOR + RLE between similar chunks (within 10% size). Zero runs encoded as [count][0x00]; diffs as [count][0x01][bytes].
Fine-tuned variants share most weights with base model — delta from base to fine-tune can be tiny.
Skipped if delta not ≥20% smaller
8 · LZSS
64 KB sliding-window dictionary; 258-byte max match; 32 768-entry hash table with 128-link chain depth. Lazy matching.
After byte shuffle, grouped exponent bytes form long repeated runs — all captured within the 64 KB window. 16× larger than v1.
Skipped if output ≥ input
9 · rANS
Order-1 context-adaptive Range ANS. PROB_SCALE=4096. Builds 256 conditional frequency tables for data >4 KB.
After LZSS, strong byte-to-byte correlations remain. Order-1 captures these for 5–15% additional reduction on structured data.
Skipped if output ≥ input

Storing a Model

Two paths: single-call store (recommended) or compress-and-inspect for custom pipelines.

Path A — Single call (recommended)

Use memalgo_store_model() — compresses and writes the .cell file in one call.

memalgo_api.hC API
// Weights already in memory (e.g. from PyTorch export)
const uint8_t* weights    = get_model_weights();
const size_t   weight_len = get_model_size();

// Store with automatic pipeline selection
int rc = memalgo_store_model(
    "models/bert-base.cell",  // output path
    weights,
    weight_len,
    NULL                   // NULL = default options
);

if (rc != MC_OK) {
    fprintf(stderr, "Store failed: %d\n", rc);
    return rc;
}
The Analyzer automatically detects float32 vs. INT4 and selects the optimal combination of Byte Shuffle, Bit-Pack, LZSS, and rANS — no configuration needed.

Path B — Compress + inspect stats

Use memalgo_compress_stats() to see metrics before writing.

memalgo_api.hC API
uint8_t* compressed = NULL;
size_t   comp_len   = 0;
McStats  stats;

int rc = memalgo_compress_stats(
    weights, weight_len,
    &compressed, &comp_len,
    &stats,
    NULL          // default options
);

printf("Original  : %zu B\n",   stats.original_size);
printf("Compressed: %zu B\n",  stats.compressed_size);
printf("Ratio     : %.1f%%\n",  stats.ratio * 100);
printf("Chunks    : %u / %u\n",
       stats.unique_chunks, stats.total_chunks);

// Only write if worthwhile
if (stats.ratio < 0.95) {
    FILE* f = fopen("model.cell", "wb");
    fwrite(compressed, 1, comp_len, f);
    fclose(f);
}
memalgo_free(compressed);

Loading & Accessing a Model

Load decompresses all chunks in parallel across available CPU cores, then returns a contiguous weight buffer bit-perfect with the original.

Single-call load

memalgo_api.hC API
uint8_t* weights = NULL;
size_t   size    = 0;

// Decompress in parallel across all CPU cores
int rc = memalgo_load_model(
    "models/bert-base.cell",
    &weights,
    &size
);

if (rc != MC_OK) {
    fprintf(stderr, "Load failed: %d\n", rc);
    return rc;
}

// weights = original bytes, bit-perfect
load_weights_into_engine(weights, size);
memalgo_free(weights);
No configuration needed — per-chunk flags written at compress time tell the decompressor exactly which stages to reverse (rANS → LZSS → Pattern → Unshuffle → Unbitpack).

Inspect without decompressing

memalgo_api.hC API
McCellInfo info;
memalgo_cell_info("models/bert-base.cell", &info);

printf("Version  : %u\n",    info.version);
printf("Orig size: %llu B\n", info.original_size);
printf("Comp size: %llu B\n", info.compressed_size);
printf("Ratio    : %.2f\n",   info.ratio);
printf("CRC32 OK : %s\n",    info.crc_valid ? "yes" : "CORRUPT");
printf("Stages   : 0x%02X\n", info.pipeline_config);
// bit0=bitpack  bit1=shuffle  bit2=pattern
// bit3=delta    bit4=lzss     bit5=rans

Model Registry

The registry (registry.idx) maps model names → .cell file paths, tracks versions, stores metadata, and supports fast lookup by name.

Create & populate

memalgo_api.hC API
// Open (creates if not exists)
McRegistry* reg = memalgo_registry_open(
    "models/registry.idx"
);

// Register a model: name, path, orig_size, type, version
memalgo_registry_add(
    reg,
    "bert-base-v1",
    "models/bert-base.cell",
    original_size,
    "transformer",
    1
);

// Register a fine-tuned variant
memalgo_registry_add(
    reg,
    "bert-sentiment-v2",
    "models/bert-sentiment.cell",
    ft_size, "transformer", 2
);

memalgo_registry_close(reg);

Look up & list models

memalgo_api.hC API
McRegistry* reg = memalgo_registry_open(
    "models/registry.idx"
);

// ── Find one model ─────────────────────
McModelInfo info;
int rc = memalgo_registry_find(
    reg, "bert-base-v1", &info
);
if (rc == MC_OK) {
    // Load directly from registry entry
    uint8_t *w; size_t sz;
    memalgo_load_model(info.cell_path, &w, &sz);
    engine_load(w, sz);
    memalgo_free(w);
}

// ── List all models ────────────────────
McModelInfo all[128]; size_t count = 0;
memalgo_registry_list(reg, all, 128, &count);
for (size_t i = 0; i < count; i++) {
    printf("[%zu] %s v%u  %.1f%% saved\n",
        i, all[i].name,
        all[i].version, all[i].ratio*100);
}
memalgo_registry_close(reg);
The registry stores paths, not file data. Moving a .cell file without updating the registry returns MC_ERR_NOT_FOUND. Use memalgo_registry_update_path() to fix stale entries.

Streaming API — Models Larger Than RAM

Feed data in 1 MB chunks — never buffers the full model in memory.

Stream compress (write)

memalgo_api.hC API
McStream* s = memalgo_stream_compress_open(
    "huge_model.cell", NULL
);

while (has_more_data()) {
    uint8_t  chunk[1024 * 1024];
    size_t   n = read_next_chunk(
                     chunk, sizeof(chunk));

    int rc = memalgo_stream_write(s, chunk, n);
    if (rc != MC_OK) {
        memalgo_stream_compress_close(s);
        return rc;
    }
}

memalgo_stream_compress_close(s);
// "huge_model.cell" is now complete

Stream decompress (read)

memalgo_api.hC API
McStream* d = memalgo_stream_decompress_open(
    "huge_model.cell"
);

uint8_t buf[1024 * 1024];
size_t  read_len = 0;

// Process layer by layer — no full-model buffer
while (
    memalgo_stream_read(
        d, buf, sizeof(buf), &read_len
    ) == MC_OK && read_len > 0
) {
    process_layer(buf, read_len);
}

memalgo_stream_decompress_close(d);
Memory usage
Only one in-flight chunk (1 MB) held in memory at a time, plus fixed-size pipeline buffers per stage. Safe for models larger than available RAM.
Thread safety
Each McStream is single-threaded. Open separate streams per thread. The registry supports concurrent reads.
Integrity
CRC32 verified per chunk during streaming. A corrupt chunk returns MC_ERR_CRC immediately — no silent corruption.

End-to-End Integration Pattern

Complete workflow from training to serving using MemCell as the model storage layer.

Step 1 · After Training train
  • Export weights to raw binary buffer
  • Call memalgo_store_model()
  • Register with version tag in local registry
  • Upload .cell file (30–60% smaller)
memalgo_store_model( "bert-v3.cell", weights, size, NULL); memalgo_registry_add( reg, "bert", "bert-v3.cell", size, "transformer", 3);
Step 2 · On Deployment deploy
  • Download .cell file (smaller transfer)
  • Verify CRC32 with memalgo_cell_info()
  • Abort if CRC fails — no silent corruption
  • Register in production registry
McCellInfo info; memalgo_cell_info( "bert-v3.cell", &info); if (!info.crc_valid) abort_deploy(); memalgo_registry_add( prod_reg, "bert", ...);
Step 3 · On Inference serve
  • Look up model in registry by name
  • Load + decompress in parallel (3–8×)
  • Pass weight pointer to inference engine
  • Call memalgo_free() after VRAM copy
McModelInfo m; memalgo_registry_find( reg, "bert", &m); uint8_t *w; size_t sz; memalgo_load_model( m.cell_path, &w, &sz); engine_load(w, sz); memalgo_free(w);

memalgo info       model.cell
